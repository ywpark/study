# 인터럽트와 시스템 콜

---

## 기본 정보

- 운영체제는 프로그램이 실행되는 동안 필요한 순간에 개입하여 자원을 관리한다.
- 이때 운영체제가 개입하는 대표적인 계기가 `인터럽트`, `예외`, `시스템 콜`이다.

## 인터럽트 (Interrupt)

- CPU가 현재 하던 일을 잠시 멈추고 다른 중요한 일을 처리하도록 만드는 신호
- 주로 하드웨어에 의해 발생
- 예시
    - 키보드 입력
    - 마우스 입력
    - 디스크 입출력 완료
    - 타이머 만료

## 인터럽트 처리 과정

1. 인터럽트 발생
2. CPU가 현재 실행 중인 작업 정보를 저장
3. 인터럽트 서비스 루틴 (Interrupt Service Routine, ISR) 실행
4. 인터럽트 처리 완료 후 원래 작업으로 복귀

## 인터럽트의 종류

- `하드웨어 인터럽트`
    - 외부 장치에 의해 발생
- `타이머 인터럽트`
    - 정해진 시간이 지나면 발생
    - 선점형 스케줄링에서 매우 중요
- `소프트웨어 인터럽트`
    - 프로그램 실행 도중 소프트웨어적으로 발생
    - 시스템 콜과 밀접하게 연결됨

## 예외 (Exception)

- CPU가 명령어를 실행하는 과정에서 비정상적인 상황이 발생했을 때 처리되는 사건
- 예시
    - 0으로 나누기
    - 잘못된 메모리 접근
    - 권한 없는 명령 실행

## 시스템 콜 (System Call)

- 사용자 프로그램이 운영체제의 기능을 사용하기 위해 호출하는 인터페이스
- 사용자 프로그램은 하드웨어에 직접 접근할 수 없으므로 시스템 콜을 통해 커널의 도움을 받아야 함
- 예시
    - 파일 열기/닫기
    - 프로세스 생성
    - 메모리 할당
    - 네트워크 통신

## 시스템 콜의 동작 과정

1. 사용자 프로그램이 시스템 콜 호출
2. CPU가 사용자 모드에서 커널 모드로 전환
3. 커널이 요청된 서비스를 수행
4. 결과 반환 후 다시 사용자 모드로 전환

## 사용자 모드와 커널 모드

- `사용자 모드`
    - 일반 프로그램이 실행되는 제한된 모드
    - 중요한 자원에 직접 접근 불가
- `커널 모드`
    - 운영체제가 실행되는 특권 모드
    - 모든 자원과 명령어 접근 가능

## 왜 이런 구조가 필요할까?

- 모든 프로그램이 하드웨어에 직접 접근하면 시스템 안정성이 크게 떨어짐
- 악성 코드나 버그가 전체 시스템을 망가뜨릴 수 있음
- 따라서 운영체제는 시스템 콜을 통해서만 안전하게 자원에 접근하도록 함

## 인터럽트와 시스템 콜의 관계

- 인터럽트는 보통 외부 사건에 의해 발생
- 시스템 콜은 프로그램이 운영체제 서비스를 요청하기 위해 의도적으로 발생시키는 소프트웨어 인터럽트 성격을 가짐
- 둘 다 CPU의 흐름을 바꾸고 커널이 개입한다는 점에서 연결됨

## 정리

- 인터럽트는 운영체제가 외부 사건에 반응하게 하고, 시스템 콜은 사용자 프로그램이 운영체제 기능을 사용하게 한다.
- 두 개념은 운영체제가 CPU 제어권을 얻는 핵심적인 방식이다.
