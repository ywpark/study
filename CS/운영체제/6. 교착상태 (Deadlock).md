# 교착상태 (Deadlock)

---

## 기본 정보

- 두 개 이상의 프로세스/스레드가 서로 상대방의 작업이 끝나기 만을 기다리면서 무한정 대기
상태에 빠지는 상황

## 교착상태 발생 조건

- 아래 4가지 조건이 모두 만족해야 교착상태 발생
- `상호 배제 (Mutual Exclusion)`
    - 자원은 한 번에 하나의 프로세스만 사용할 수 있어야 함
- `점유와 대기 (Hold and Wait)`
    - 프로세스가 자원을 하나 이상 점유한 상태에서 다른 자원을 기다림
- `비선점 (No Preemption)`
    - 다른 프로세스가 점유 중인 자원을 강제로 뺏을 수 없음
- `원형 대기 (Circular Wait)`
    - 자원을 기다리는 프로세스들이 원형으로 연결되어 있음
        - P1 → P2 → P3 → P1 형태로 자원을 기다리는 상황

## 교착 상태 해결 방법

- `예방 (Prevention)`
    - 교착 상태가 발생하지 않도록 함
    - 교착 상태가 발생하지 않음은 보장할 수 있으나 부작용이 따르는 존
    - 교착 상태 발생 조건 중 하나를 깨트리도록 설계
        - `상호 배제`
            - 자원 공유 가능하도록 설계
            - 이론적으로는 가능하나 현실적으로는 불가능 함
        - `점유와 대기`
            - 모든 자원을 한 번에 요청하여 모두 할당을 받거나 아예 할당하지 않는 방식
            으로 규칙 설정
            - 자원의 활용률이 낮아짐
        - `비선점`
            - 선점이 가능한 자원(CPU, 메모리)에 한해는 효과적이나 모든 자원이 선점 가능한 것은 아님
            - 범용적으로 활용은 불가능
        - `원형대기`
            - 자원에 번호를 붙이고 오름차순으로 할당하면 원형 대기는 발생하지 않음
            - 자원에 번호를 붙이는 것은 어려운 작업
            - 어떤 자원에 어떤 번호를 붙이느냐에 따라 활용률이 달라짐
- `회피 (Avoidance)`
    - 교착 상태가 무분별한 자원 할당으로 인해 발생한 것으로 가정
    - 배분할 수 있는 자원의 양을 고려하여 교착 상태가 발생하지 않을 만큼만 자원 배분
    - `안전 순서열 (Safe Sequence)`
        - 프로세스들의 실행 순서를 나타내는 시퀀스로, 이 순서대로 자원을 할당하면 모든
        프로세스가 교착상태 없이 완료
    - `안전 상태 (Safe State)`
        - 교착 상태 없이 모든 프로세스가 자원을 할당 받고 종료될 수 있는 상태
        - 안전 순서열이 있는 상태
    - `불안전 상태 (Unsafe State)`
        - 교착 상태가 아직 발생하지 않았지만, 앞으로의 요청에 따라 교착 상태가 발생할 수도
        있는 상태
        - 안전 순서열이 없는 상태
    - 안전 상태 → 안전 상태 인 경우에만 자원을 할당하는 방식
    - 대표적으로 `은행원 알고리즘 (Banker’s Algorithm)`
- `탐지 및 회복 (Detection & Recovery)`
    - 교착 상태가 발생했는지 탐지 후, 강제로 해제
    - `선점을 통한 회복`
        - 교착 상태가 해결될 때까지 한 프로세스씩 자원을 몰아주는 방식
    - `프로세스 강제 종료를 통한 회복`
        - 교착 상태에 놓인 프로세스 모두 강제 종료
            - 작업 내역을 잃을 위험
        - 교착 상태가 해결될 때까지 한 프로세스씩 강제 종료
            - 오버헤드 발생
- `무시 ( Ignore)`
    - 운영체제 수준에서 아무런 특별한 조치를 취하지 않는 전략
    - `타조 알고리즘 (Ostrich Algorithm)`
        - 문제를 감지하거나 해결하려고 노력하기보다는 문제 발생을 무시하거나 묵인하는 전략
