# 운영체제의 기본

---

## 운영체제가 필요한 이유

- 컴퓨터는 `중앙처리장치 (Central Processing Unit, CPU)`, 메모리, 디스크, 입출력장치처럼 여러 하드웨어 자원을 가지고 있다.
- 여러 프로그램이 동시에 실행되면 한정된 자원을 누가, 언제, 얼마나 사용할지 결정해야 한다.
- 운영체제는 이러한 자원 사용을 조정하고, 프로그램이 하드웨어를 직접 다루지 않아도 되도록 추상화된 기능을 제공한다.
- 즉, 운영체제는 `자원 관리`, `추상화`, `보호`, `편의성`을 제공하는 핵심 시스템 소프트웨어이다.

## 운영체제의 주요 역할

- `자원 관리`
  - CPU, 메모리, 파일, 입출력장치를 효율적으로 관리한다.
- `추상화`
  - 복잡한 하드웨어를 파일, 프로세스, 가상 메모리 같은 개념으로 단순화하여 제공한다.
- `보호`
  - 하나의 프로그램이 시스템 전체를 망가뜨리지 못하도록 접근을 제한한다.
- `편의성`
  - 프로그램이 실행, 저장, 통신할 수 있는 표준 인터페이스를 제공한다.

## 커널과 사용자 영역

- `커널 (Kernel)`
  - 운영체제의 핵심 부분으로, 자원 관리와 하드웨어 제어를 담당한다.
- `사용자 영역 (User Space)`
  - 일반 응용 프로그램이 실행되는 영역이다.
- 응용 프로그램이 하드웨어에 직접 접근하지 못하도록 사용자 영역과 커널 영역을 분리한다.
- 이 분리 덕분에 잘못된 프로그램이 전체 시스템을 쉽게 손상시키지 못한다.

## 이중 모드 (Dual Mode)

- CPU는 명령어를 실행할 때 `사용자 모드`와 `커널 모드`를 구분한다.
- `사용자 모드`
  - 일반 프로그램이 실행되는 모드이다.
  - 중요한 자원 접근이나 특권 명령 실행이 제한된다.
- `커널 모드`
  - 운영체제가 실행되는 모드이다.
  - 메모리 관리, 장치 제어, 인터럽트 처리 같은 특권 작업이 가능하다.
- 이중 모드는 운영체제가 시스템을 보호하기 위한 기본 장치이다.

## 인터럽트와 예외

- `인터럽트 (Interrupt)`
  - 하드웨어가 CPU에 처리가 필요한 사건이 발생했음을 알리는 신호이다.
  - 예를 들어 키보드 입력, 디스크 입출력 완료가 있다.
- `예외 (Exception)`
  - CPU가 명령어를 실행하는 도중 발생하는 비정상 상황이다.
  - 예를 들어 0으로 나누기, 잘못된 메모리 접근이 있다.
- 운영체제는 인터럽트와 예외를 처리하면서 시스템의 안정성을 유지한다.

## 시스템 콜 (System Call)

- 응용 프로그램이 운영체제의 기능을 사용하기 위해 요청하는 인터페이스이다.
- 파일 읽기, 프로세스 생성, 메모리 할당 같은 작업은 시스템 콜을 통해 수행된다.
- 시스템 콜이 호출되면 CPU는 사용자 모드에서 커널 모드로 전환된다.

## 시스템 콜의 동작 과정

1. 사용자 프로그램이 시스템 콜을 요청한다.
2. CPU가 소프트웨어 인터럽트를 통해 커널 모드로 전환한다.
3. 운영체제가 요청 내용을 검사하고 필요한 작업을 수행한다.
4. 작업 결과를 사용자 프로그램에 반환한다.
5. 다시 사용자 모드로 전환하여 프로그램 실행을 이어간다.

## 운영체제가 제공하는 추상화

- `프로세스`
  - 실행 중인 프로그램이라는 추상화
- `가상 메모리`
  - 물리 메모리보다 더 큰 메모리를 쓰는 것처럼 보이게 하는 추상화
- `파일`
  - 디스크의 복잡한 구조를 숨기고 데이터를 논리적으로 다루게 해주는 추상화
- `소켓`
  - 네트워크 통신을 단순한 인터페이스로 제공하는 추상화

## 정리

- 운영체제는 단순히 프로그램을 실행시키는 소프트웨어가 아니라, 컴퓨터 시스템 전체를 안전하고 효율적으로 운영하기 위한 관리자이다.
- 사용자 프로그램은 운영체제가 제공하는 추상화와 시스템 콜을 통해 하드웨어를 간접적으로 사용한다.
