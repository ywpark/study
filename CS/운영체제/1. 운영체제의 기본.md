# 운영체제의 기본

---

## 운영체제 (Operating System, OS)

- 사용자의 하드웨어, 시스템 리소스를 제어하고 프로그램에 대한 일반적 서비스를
지원하는 시스템 소프트웨어

## 기본 정보

- 실행할 프로그램에 필요한 자원을 할당
- `커널 (Kernel)`
    - 운영체제의 핵심 서비스를 담당하는 부분
- 응용프로그램은 자원에 직접 접근 안됨
    - 자원에 접근하려면 OS 를 통해서만 접근하도록 하여 자원을 보호
- 서비스
    - `프로세스 관리 (Process Management)`
        - 메모리에서 실행 중인 프로그램을 관리
        - 프로세스의 생성/실행/삭제 등 여러 프로세스들을 관리
    - `주기억장치 관리 (Main Memory Management)`
        - 프로세스에게 메모리 공간 할당/회수 등을 관리
        - `가상메모리` : 물리적 실제 메모리보다 큰 용량을 갖도록 함
    - `파일 관리 (File Management)`
        - sector/track 으로 구성된 디스크를 파일이라는 논리적 관점으로 관리
        - 파일 과 디렉토리의 생성/삭제를 관리
    - `보조기억장치 관리 (Secondary Storage Management)`
        - 디스크 공간 관리
    - `입출력 장치 관리 (I/O Device Management)`

## 이중 모드 (Dual Mode)

- CPU가 명령어를 실행하는 모드를 구분
    - `사용자 모드 (User Mode)`
        - 운영체제의 서비스를 제공받을 수 없는 실행 모드
        - 커널 영역의 코드를 실행할 수 없는 실행 모드
        - 자원 접근 불가
    - `커널 모드 (Kernel Mode)`
        - 운영체제의 서비스를 제공받을 수 있는 실행 모드
        - 자원 접근 및 모든 명령어 실행 가

## 시스템 콜 (System Calls)

- 커널 모드로 전환하여 실행하기 위해 호출
- 소프트웨어 인터럽트
    - 시스템 콜이 처리되는 방식은 하드웨어 인터럽트 처리 방식과 유사
- 운영체제 마다 정해진 시스템 호출 함수가 존재
