# 입출력과 디스크 스케줄링

---

## 기본 정보

- 운영체제는 CPU와 메모리뿐 아니라 입출력장치도 관리한다.
- 입출력장치는 CPU보다 훨씬 느린 경우가 많기 때문에 효율적인 관리가 중요하다.

## 입출력 장치와 장치 컨트롤러

- `입출력 장치`
    - 키보드, 마우스, 디스크, 프린터, 네트워크 카드 등
- `장치 컨트롤러`
    - 장치를 제어하는 하드웨어 구성 요소
    - CPU와 장치 사이의 중간 역할 수행
- 운영체제는 장치 드라이버를 통해 장치 컨트롤러와 상호작용한다.

## 장치 드라이버

- 운영체제와 하드웨어 장치를 연결해주는 소프트웨어
- 장치마다 동작 방식이 다르기 때문에 장치별 드라이버가 필요

## 입출력 처리 방식

- `프로그램에 의한 입출력`
    - CPU가 직접 장치 상태를 계속 확인하며 처리
    - 비효율적
- `인터럽트 기반 입출력`
    - 입출력 완료 시 장치가 CPU에 인터럽트를 보내는 방식
    - CPU 낭비를 줄일 수 있음
- `직접 메모리 접근 (Direct Memory Access, DMA)`
  - CPU가 아닌 DMA 컨트롤러가 메모리와 장치 사이의 데이터 전송 담당
    - 대량 데이터 전송 시 효율적

## 버퍼링, 캐싱, 스풀링

- `버퍼링 (Buffering)`
    - 데이터 전송 속도 차이를 완화하기 위해 임시 저장 공간 사용
- `캐싱 (Caching)`
    - 자주 사용하는 데이터를 빠른 저장장치에 저장해 성능 향상
- `스풀링 (Spooling)`
    - 여러 작업을 한꺼번에 모아 순서대로 처리
    - 프린터 작업이 대표적 예시

## 디스크 스케줄링이 필요한 이유

- 디스크는 데이터를 읽기 위해 헤드를 이동시켜야 함
- 요청 순서를 어떻게 정하느냐에 따라 전체 성능이 크게 달라짐
- 목표는 `탐색 시간(Seek Time)`을 줄이는 것

## 대표적인 디스크 스케줄링 알고리즘

- `FCFS (First Come First Served)`
    - 요청이 들어온 순서대로 처리
    - 공정하지만 비효율적일 수 있음

- `SSTF (Shortest Seek Time First)`
    - 현재 헤드 위치에서 가장 가까운 요청을 먼저 처리
    - 평균 탐색 시간 감소
    - 먼 요청이 오래 기다릴 수 있음

- `SCAN`
    - 헤드가 한 방향으로 이동하며 경로상의 요청을 처리
    - 끝까지 갔다가 반대 방향으로 이동
    - 엘리베이터 알고리즘이라고도 불림

- `C-SCAN (Circular SCAN)`
    - 한 방향으로만 요청 처리
    - 끝에 도달하면 처음 위치로 빠르게 이동 후 다시 같은 방향으로 처리
    - 대기 시간의 편차를 줄이는 데 유리

## 디스크 스케줄링 알고리즘 비교

- `FCFS (First Come First Served)`
    - 단순하고 공정
    - 평균 성능은 낮을 수 있음
- `SSTF`
    - 성능이 좋을 수 있음
    - 기아 현상 가능
- `SCAN`
    - 전체적으로 균형 잡힌 성능
- `C-SCAN`
    - 요청 대기 시간을 더 균등하게 관리

## 정리

- 입출력 장치는 CPU보다 느리므로 운영체제는 인터럽트, DMA, 버퍼링 같은 기법으로 성능을 높인다.
- 디스크 스케줄링은 디스크 접근 순서를 최적화하여 전체 시스템 성능을 향상시키는 핵심 기법이다.
