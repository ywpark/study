# 페이지 교체와 프레임할당

---

## 요구 페이징 (Demand Paging)

- 프로그램 실행에 필요한 모든 페이지를 처음부터 메모리에 올리지 않고, 실제로 참조되는 순간에만 적재하는 방식이다.
- 메모리 사용량을 줄일 수 있지만, 필요한 페이지가 없으면 페이지 폴트가 발생한다.

## 페이지 교체가 필요한 이유

- 메모리가 가득 찬 상태에서 새로운 페이지를 적재하려면 기존 페이지 중 하나를 내보내야 한다.
- 어떤 페이지를 내보내느냐에 따라 페이지 폴트 수와 시스템 성능이 크게 달라진다.

## 페이지 참조열 (Page Reference String)

- CPU가 시간 순서대로 참조한 페이지 번호의 나열이다.
- 페이지 교체 알고리즘의 성능을 비교할 때 사용한다.

## 대표적인 페이지 교체 알고리즘

### FIFO (First-In, First-Out)

- 가장 먼저 들어온 페이지를 먼저 내보낸다.
- 구현은 쉽지만 자주 쓰이는 오래된 페이지도 교체할 수 있다.

### Second Chance

- FIFO를 개선한 방식이다.
- 참조 비트를 확인해 최근에 사용된 페이지에 한 번 더 기회를 준다.

### OPT (Optimal Page Replacement)

- 앞으로 가장 오랫동안 사용되지 않을 페이지를 교체한다.
- 이론적으로 가장 좋은 성능을 보이지만 미래를 알아야 하므로 실제 구현은 어렵다.

### LRU (Least Recently Used)

- 가장 오랫동안 참조되지 않은 페이지를 교체한다.
- 실제 OPT를 근사하는 대표적인 방식이다.

## Belady의 anomaly

- 일반적으로 프레임 수가 늘어나면 페이지 폴트는 줄어들 것 같지만, FIFO에서는 오히려 페이지 폴트가 증가할 수 있다.
- 이를 `Belady의 anomaly`라고 한다.

## 지역성 (Locality)

- 프로그램은 모든 메모리를 무작위로 사용하는 것이 아니라, 특정 영역을 집중적으로 참조하는 경향이 있다.
- `시간 지역성`
  - 최근 사용한 데이터를 다시 사용할 가능성이 높다.
- `공간 지역성`
  - 최근 접근한 위치 주변을 다시 사용할 가능성이 높다.
- LRU나 Working Set 같은 개념은 이러한 지역성을 기반으로 한다.

## 스레싱 (Thrashing)

- 프로세스가 실제 작업보다 페이지 교체에 더 많은 시간을 쓰는 현상이다.
- 보통 각 프로세스에 충분한 프레임이 주어지지 않을 때 발생한다.
- CPU 사용률이 떨어지고 시스템 전체 성능이 급격히 나빠진다.

## 프레임 할당 방식

- 운영체제는 각 프로세스에 몇 개의 프레임을 줄지 결정해야 한다.
- `정적 할당`
  - 실행 중 상황 변화와 무관하게 미리 정한 기준으로 할당
- `동적 할당`
  - 페이지 폴트 빈도나 최근 참조 패턴을 보며 조절

## Working Set Model

- 최근 일정 시간 동안 실제로 참조된 페이지 집합을 기준으로 필요한 프레임 수를 추정한다.
- 프로세스가 현재 활발히 사용하는 페이지들을 메모리에 유지하도록 돕는다.

## 페이지 폴트 빈도 (Page-Fault Frequency, PFF)

- 페이지 폴트 빈도가 너무 높으면 프레임을 더 주고, 너무 낮으면 일부 프레임을 회수하는 방식이다.
- 동적으로 프레임 수를 조절하여 스레싱을 완화한다.

## 정리

- 페이지 교체와 프레임 할당은 요구 페이징이 안정적으로 동작하기 위한 핵심 요소이다.
- 좋은 알고리즘은 페이지 폴트를 줄이고, 스레싱을 막으며, 지역성을 잘 활용해야 한다.
