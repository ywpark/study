# 페이지 교체와 프레임할당

---

## 요구 페이징 (Demand Paging)

- 처음부터 모든 페이지를 적재하지 않고 필요한 페이지만을 메모리에 적재하는 기법
    - 요구되는 페이지만 적재하는 기법
- 과정
    1. CPU가 특정 페이지에 접근하는 명령어를 실행
    2. 해당 페이지가 현재 메모리에 있을 경우 (유효비트 1) CPU는 페이지가 적재된
    프레임에 접근
    3. 해당 페이지가 현재 메모리에 없을 경우 (유효비트 0) 페이지 폴트가 발생
    4. 페이지 폴트 처리 루틴은 해당 페이지를 메모리로 적재하고 유효 비트를 1로 설정
    5. 다시 1번을 수행
- `안정적으로 작동하려면 아래의 두 가지를 해결해야 함`
    - `페이지 교체`
    - `프레임 할당`

## 페이지 교체

- OS 는 기존에 적재된 불필요한 페이지를 선별해 보조기억장치로 내보냄
    - 메모리가 가득 찬 상황에서 당장 실행에 필요한 페이지를 적재하려고 할 때
- `페이지 교체 알고리즘`
    - 기존에 적재된 페이지 중에 어떤 페이지를 내보낼 것인가
        - 페이지 폴트의 횟수가 가장 적은 알고리즘이 선능이 좋다고 판단
        - `페이지 폴트가 많다?`
            - 보조기억장치에 접근하는 횟수가 많다
    - `페이지 폴트 횟수를 어떻게 알 수 있나?`
        - `페이지 참조열 (Page Reference String)`
            - CPU가 참조하는 페이지들 중 연속된 페이지를 생략한 페이지열
            - `연속된 페이지를 생략하는 이유?`
                - 이미 메모리로 페이지를 로드를 하고 난 뒤에는 동일한 페이지에서는
                페이지 폴트가 발생하지 않기 때문
    - `FIFO (First-In First-Out) 페이지 교체 알고리즘`
        - 가장 단순한 방식
        - 메모리에 가장 먼저 적재된 페이지부터 교체되는 알고리즘
        - 프로그램 실행 내내 사용될 페이지조차 오래되면 교체되기 때문에 페이지 폴트가
        빈번하게 일어날 수 있음
    - `2차 기회 (Second Chance) 페이지 교체 알고리즘`
        - FIFO 페이지 교체 알고리즘의 단점을 보완하기 위한 기법
        - 페이지 참조 비트 와 적재 시간으로 관리
        - 메모리에 오래 적재되었더라도 CPU가 최근에 접근한 적이 있다면 교체를 미룸
    - `최적 페이지 교체 알고리즘`
        - CPU에 의해 참조되는 횟수를 고려하는 페이지 교체 알고리즘
        - 사용 빈도가 가장 낮은 페이지를 예측하여 해당 페이지를 교체
        - 예측하는 것이 까다롭기 때문에 실제 구현이 매우 어려움
        - 주로 다른 페이지 교체 알고리즘의 이론상 성능을 평가하기 위한 목적으로 사용
    - `LRU (Least Recently Used) 페이지 교체 알고리즘`
        - 가장 오랫동안 사용되지 않은 페이지를 교체하는 알고리즘
        - 페이지마다 마지막으로 사용한 시간을 토대로 가장 사용이 적었던 페이지를 교체
    - 위에 4가지 이외에도 페이지 교체 알고리즘이 존재함

## 스레싱 (Thrashing)

- 프로세스가 실제 실행되는 시간보다 페이징에 더 많은 시간을 소요하여 성능이 저해되는 문제
    - 각 프로세스가 필요로 하는 프레임 수가 보장이 안됨
- `페이지 폴트가 자주 발생하는 이유가 멀까?`
    - 나쁜 페이지 교체 알고리즘을 사용하기 때문
    - 프로세스가 사용할 수 있는 프레임 자체가 적어서
        - 물리 메모리 용량이 적음
- CPU의 성능이 뛰어나도 동시에 실행되는 프로세스를 수용할 물리적 메모리 용량이 적으면
전체 컴퓨터의 성능이 나빠짐
- `OS 는 각 프로세스들이 무리 없이 실행하기 위한 최소한의 프레임 수를 파악하고 프로세스들에게
적절한 수만큼 프레임을 할당`

## 프레임 할당

- OS 는 프로세스에 적절한 수의 프레임을 할당해야 함
- 정적 할당 방식
    - 프로세스의 크기와 물리 메모리의 크기만 고려한 방식
    - 하나의 프로세스가 실제로 얼마나 많은 프레임이 필요할지는 결국 실행해 봐야 아는 경우가
    많음
    - `균등 할당 (Equal Allocation)`
        - 모든 프로세스들에게 균등하게 프레임을 할당하는 방식
        - 프로세스들의 크기가 각기 다른데 모두 동일하게 할당
    - `비례 할당 (Proportonal Allocation)`
        - 프로세스의 크기에 비례하여 프레임을 할당하는 방식
- 동적 할당 방식
    - `작업 집합 모델 (Working Set Model)`
        - 프로세스가 일정 기간 동안 참조한 페이지 집합을 기억하여 빈번한 페이지 교체를
        방지하는 방식
        - 작업 집합 이란?
            - 실행 중인 프로세스가 일정 시간동안 참조한 페이지의 집합
    - `페이지 폴트 빈도 (Page-Fault Frequency, PFF)`
        - 페이지 폴트의 빈도수의 상한선과 하한선을 정하여 상한선이면 프레임을 더 할당하고
        하한선이면 프레임을 회수하는 방식
