# CPU 스케줄링

---

## 중앙처리장치 스케줄링이 필요한 이유

- 여러 프로세스가 동시에 실행되더라도 실제 `중앙처리장치 (Central Processing Unit, CPU)` 코어 수는 제한적이다.
- 운영체제는 어떤 프로세스에 CPU를 먼저 줄지 결정해야 한다.
- 이 결정 과정이 CPU 스케줄링이다.

## CPU burst와 I/O burst

- 프로세스는 실행 과정에서 `CPU 버스트 (CPU burst)`와 `입출력 버스트 (Input/Output burst, I/O burst)`를 반복한다.
- `CPU 버스트 (CPU burst)`
  - CPU를 사용하여 계산을 수행하는 구간
- `입출력 버스트 (Input/Output burst, I/O burst)`
  - 입출력 작업을 기다리는 구간
- 입출력 위주의 프로세스는 CPU를 짧게 자주 사용하고, 계산 위주의 프로세스는 CPU를 길게 사용한다.
- 운영체제는 이런 특성을 고려해 스케줄링하면 전체 효율을 높일 수 있다.

## 스케줄링 목표와 평가 지표

- `중앙처리장치 이용률 (CPU utilization)`
  - CPU가 가능한 한 놀지 않도록 하는 것
- `처리량 (Throughput)`
  - 단위 시간당 완료되는 작업 수
- `반환 시간 (Turnaround time)`
  - 작업 제출부터 완료까지 걸린 총 시간
- `대기 시간 (Waiting time)`
  - 준비 큐에서 대기한 시간
- `응답 시간 (Response time)`
  - 요청 후 첫 응답까지 걸린 시간

## 선점형과 비선점형

- `선점형 스케줄링`
  - 실행 중인 프로세스로부터 CPU를 빼앗을 수 있다.
  - 응답성이 좋지만 문맥 교환 오버헤드가 있다.
- `비선점형 스케줄링`
  - 실행 중인 프로세스가 스스로 CPU를 반납할 때까지 기다린다.
  - 오버헤드는 적지만 반응성이 떨어질 수 있다.

## 대표적인 알고리즘

### FCFS (First Come First Served)

- 먼저 온 프로세스를 먼저 실행한다.
- 단순하지만 긴 작업이 앞에 오면 뒤의 짧은 작업이 오래 기다릴 수 있다.
- 이를 `호위 효과 (Convoy Effect)`라고 한다.

### SJF (Shortest Job First)

- CPU 사용 시간이 가장 짧은 작업을 먼저 실행한다.
- 평균 대기 시간을 줄이는 데 유리하다.
- 하지만 실제로 다음 CPU burst 길이를 정확히 예측하기 어렵다.

### SRT (Shortest Remaining Time)

- SJF의 선점형 버전이다.
- 남은 작업 시간이 가장 짧은 프로세스를 우선 실행한다.

### RR (Round Robin)

- 각 프로세스에 일정한 `타임 슬라이스`를 부여하고 돌아가며 CPU를 준다.
- 시분할 시스템에 적합하며 응답성이 좋다.
- 타임 슬라이스가 너무 크면 FCFS와 비슷해지고, 너무 작으면 문맥 교환이 많아진다.

### 우선순위 스케줄링 (Priority Scheduling)

- 우선순위가 높은 프로세스를 먼저 실행한다.
- 우선순위가 낮은 프로세스는 계속 밀릴 수 있는데, 이를 `기아 (Starvation)`라고 한다.
- 이를 방지하기 위해 오래 기다린 프로세스의 우선순위를 높이는 `Aging`을 사용한다.

### 다단계 큐 / 다단계 피드백 큐 (Multilevel Queue / Multilevel Feedback Queue)

- 프로세스 특성에 따라 여러 큐로 나누어 관리하는 방식이다.
- `Multilevel Queue`
  - 큐 간 이동이 불가능하다.
- `Multilevel Feedback Queue`
  - 큐 간 이동이 가능하다.
  - 현대 운영체제의 스케줄링 개념을 설명할 때 자주 등장한다.

## 알고리즘 비교 관점

- `공정성`
  - 모든 프로세스가 적절히 CPU를 배분받는가
- `응답성`
  - 사용자 요청에 빠르게 반응하는가
- `오버헤드`
  - 문맥 교환 비용이 큰가
- `예측 가능성`
  - 실행 시간이 일정하게 관리되는가

## 정리

- CPU 스케줄링은 단순히 순서를 정하는 작업이 아니라, 시스템 전체 성능과 사용자 경험을 좌우하는 핵심 기능이다.
- 각 알고리즘은 장단점이 다르며, 운영체제는 상황에 따라 적절한 정책을 선택하거나 조합한다.
