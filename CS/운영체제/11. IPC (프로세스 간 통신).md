# IPC (프로세스 간 통신)

---

## 기본 정보

- `프로세스 간 통신 (Inter-Process Communication, IPC)`
    - 서로 다른 프로세스가 데이터를 주고받거나 동작을 조율하는 방법
- 프로세스는 기본적으로 서로 독립된 메모리 공간을 가지기 때문에 직접 데이터를 공유할 수 없음
- 따라서 운영체제가 제공하는 IPC 기법이 필요

## IPC 가 필요한 이유

- 프로세스 간 데이터 교환
- 프로세스 간 작업 분담
- 프로세스 간 실행 순서 제어
- 멀티프로세스 구조의 프로그램 구현

## 대표적인 IPC 방식

- `파이프 (Pipe)`
    - 한 프로세스의 출력이 다른 프로세스의 입력으로 전달되는 방식
    - 일반적으로 단방향 통신
    - 부모 프로세스와 자식 프로세스 사이에서 자주 사용
- `이름 있는 파이프 (Named Pipe, FIFO)`
    - 관련 없는 프로세스 사이에서도 사용할 수 있는 파이프
    - 파일 시스템에 이름이 있는 특수 파일 형태로 존재

- `메시지 큐 (Message Queue)`
    - 운영체제가 관리하는 큐를 통해 메시지를 주고받는 방식
    - 메시지 단위로 통신 가능
    - 비동기적인 통신에 적합

- `공유 메모리 (Shared Memory)`
    - 여러 프로세스가 같은 메모리 영역을 공유하는 방식
    - 매우 빠르지만 동기화가 반드시 필요
    - 공유 메모리 자체는 통신 방법이고, 데이터 충돌 방지는 세마포어/뮤텍스 같은 동기화 기법이 담당

- `소켓 (Socket)`
    - 같은 컴퓨터 내부뿐 아니라 네트워크로 연결된 다른 컴퓨터와도 통신 가능한 방식
    - 클라이언트-서버 구조에서 널리 사용

- `시그널 (Signal)`
    - 특정 이벤트가 발생했음을 간단히 알리는 방식
    - 데이터 자체를 많이 전달하기보다는 알림이나 제어 목적이 강함

## IPC 방식 비교

- `파이프`
    - 구현이 단순
    - 부모-자식 관계에서 자주 사용
- `메시지 큐`
    - 메시지 단위 관리가 편리
    - 운영체제가 큐를 관리
- `공유 메모리`
    - 가장 빠른 편
    - 동기화 문제가 직접적으로 발생
- `소켓`
    - 네트워크 통신 가능
    - 범용성이 높음
- `시그널`
    - 단순 알림에 적합
    - 대량 데이터 전달에는 부적합

## IPC 와 동기화

- IPC 는 데이터를 주고받는 방법이고, 동기화는 공유 자원을 안전하게 사용하는 방법이다.
- 특히 공유 메모리를 사용할 때는 동기화가 함께 필요하다.
- 즉, `통신`과 `동기화`는 서로 다른 문제를 해결하지만 자주 함께 등장한다.

## 정리

- 프로세스는 기본적으로 독립적이기 때문에 협업을 위해 IPC 가 필요하다.
- 상황에 따라 단순한 파이프, 빠른 공유 메모리, 범용적인 소켓 등 적절한 방식이 선택된다.
