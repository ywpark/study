# 파일시스템

---

## 파일시스템이란 무엇인가

- 파일시스템은 운영체제가 파일과 디렉터리를 저장장치에 조직하고 관리하는 방식이다.
- 사용자는 파일 이름만 보지만, 내부적으로는 블록 배치, 메타데이터 관리, 접근 권한 제어가 함께 이루어진다.

## 파일과 메타데이터

- `파일`
  - 의미 있는 데이터의 논리적 집합
- `메타데이터`
  - 파일 자체가 아닌 파일에 대한 정보
  - 예: 이름, 크기, 생성 시간, 수정 시간, 권한, 소유자, 저장 위치

## 디렉터리와 경로

- 디렉터리는 파일과 다른 디렉터리를 묶어 관리하는 특별한 파일이다.
- 파일시스템은 보통 트리 구조를 사용한다.
- `절대 경로`
  - 루트 디렉터리부터 시작하는 경로
- `상대 경로`
  - 현재 위치를 기준으로 한 경로

## 디렉터리 엔트리

- 디렉터리 내부에는 파일 이름과 해당 파일을 찾기 위한 정보가 저장된다.
- 운영체제는 디렉터리 엔트리를 통해 파일의 실제 위치나 inode를 찾는다.

## 파티션과 포매팅

- `파티션`
  - 저장장치를 논리적으로 나눈 영역
- `포매팅`
  - 특정 파일시스템 구조를 저장장치에 설정하는 작업

## 파일 할당 방식

### 연속 할당

- 파일을 연속된 블록에 저장한다.
- 순차 접근과 임의 접근이 빠르지만 외부 단편화 문제가 있다.

### 연결 할당

- 각 블록이 다음 블록의 위치를 가리키는 방식이다.
- 외부 단편화는 줄지만 임의 접근이 불리하다.

### 색인 할당

- 파일의 블록 주소를 별도의 색인 블록에 모아 관리한다.
- 임의 접근이 쉽고 구조적으로 유연하다.

## FAT (File Allocation Table) 파일 시스템

- 연결 할당 기반이지만, 다음 블록 정보를 FAT라는 별도 테이블에 모아 관리한다.
- 연결 할당의 단점을 일부 개선한 구조이다.

## 유닉스 파일 시스템과 inode

- 유닉스 파일 시스템은 inode를 통해 파일 메타데이터와 블록 위치를 관리한다.
- inode에는 직접 블록 주소와 간접 블록 주소가 들어간다.
- 이 구조 덕분에 작은 파일과 큰 파일을 모두 효율적으로 다룰 수 있다.

## 파일 접근 권한과 보호

- 파일은 아무나 읽고 쓰게 둘 수 없기 때문에 접근 권한이 필요하다.
- 일반적으로 읽기, 쓰기, 실행 권한을 사용자, 그룹, 기타 사용자 기준으로 관리한다.
- 파일 보호는 운영체제의 보안과도 연결된다.

## 캐시와 성능

- 디스크는 메모리보다 매우 느리므로 운영체제는 자주 접근하는 블록을 메모리에 캐시한다.
- 캐시 덕분에 파일 읽기와 쓰기 성능이 크게 향상된다.
- 대표적으로 버퍼 캐시나 페이지 캐시 개념이 사용된다.

## 정리

- 파일시스템은 단순히 파일을 저장하는 기능이 아니라, 저장 위치 관리, 메타데이터 관리, 권한 제어, 성능 최적화를 모두 포함하는 운영체제의 핵심 구성 요소이다.
- 연속 할당, 연결 할당, 색인 할당의 차이와 FAT, inode 구조를 함께 이해하면 파일시스템의 큰 그림을 잡기 쉽다.
