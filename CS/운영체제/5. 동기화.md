# 동기화

---

## 기본 정보

- 특정 자원에 접근할 때 한 개의 프로세스만 접근하게 하거나 프로세스를 올바른 순서대로
실행하게 하는 것
- 프로세스 동기화란?
    - 프로세스들 사이의 수행시기를 맞춤
    - `실행 순서 제어`
        - 프로세스를 올바른 순서대로 실행하기
    - `상호 배제`
        - 동시에 접근해서는 안되는 자원에 하나의 프로세만 접근하게 하기

## 실행 순서 제어를 위한 동기화

- `Reader-Writer Problem`
    - 여러 프로세스가 공유 자원에 접근할 때 발생할 수 있는 문제
    - `Writer`
        - 쓰기 작업을 하는 프로세스
            - 데이터가 변경이 되기 때문에 동시에 접근 하면 안됨
    - `Reader`
        - 읽기 작업을 하는 프로세스
            - 동시에 접근 가능
    - 실행 순서가 있기 때문에 두 개의 프로세스가 아무렇게나 실행되어서는 안됨

## 상호 배제를 위한 동기화

- `Bank Account Problem`
    - 두 개 이상의 프로세스/스레드가 같은 계좌의 잔액을 동시에 읽고, 입금/출금
    작업을 수행하는 경우
    - 공유가 불가능한 자원의 동시 사용을 피하기 위함
- `Producer & Consumer Problem`
    - 공유 자원에 `Producer` 와 `Consumer` 가 있는 상황에서 동기화가 되지 않은
    상황에서는 공유 자원에 대해서 어떠한 결과가 나올지 예측이 안됨
- `공유 자원과 임계 구역 Problem`
    - `공유 자원 (Shared Resource)`
        - 여러 프로세스 /스레드가 공유하는 자원
    - `임계 구역 (Critical Section)`
        - 동시에 실행하면 문제가 발생하는 자원에 접근하는 코드 영역
        - 임계 구역에 진입하고자 하면 진입한 프로세스 이외에는 대기해야 함
- `경쟁 조건 (Race Condition)`
    - 임계 구역에 동시에 접근하면 자원의 일관성이 깨질 수 있음

## 왜 이런 문제가 발생할까?

- 고급언어 1줄의 코드는 저급언어 여러 줄의 코드로 변환되기 때문에 실행 도중에
문맥 교환이 일어 날 수 있기 때문에 발생

## 운영체제가 임계구역을 해결하는 세가지 원칙

- `상호 배제 (Mutual Exclusion)`
    - 프로세스가 임계 구역에 진입했다면 다른 프로세스는 들어올 수 없음
- `진행 (Progress)`
    - 임계 구역에 프로세스가 진입하지 않았다면 진입하고자 하는 프로세스는
    들어갈 수 있어야 함
- `유한 대기 (Bounded Waiting)`
    - 프로세스가 임계 구역에 진입하고 싶다면 언젠가는 임계 구역에 들어갈 수
    있어야 함
        - 임계 구역에 들어가기 위해 무한정 대기는 안됨

## 바쁜 대기 (Busy Wait)

- 임계 구역이 LOCK이 걸려있는지 무한정 확인하는 상황

## 동기화 기법

- `뮤텍스 락 (Mutex Lock)`
    - 상호 배제를 위한 동기화 기법
    - `공유된 자원에서 동기화 대상이 하나인 경우`
    - `바쁜 대기 발생`
    - 구현 방법
        - `좌물쇠 역할`
            - 프로세스들이 공유하는 전역 변수 lock
        - `임계 구역을 잠그는 역할`
            - acquire 함수
        - `임계 구역의 잠금을 해제하는 역할`
            - release 함수

- `세마포 (Semaphore)`
    - Mutex Lock 보다 일반화된 방식의 동기화 기법
    - `공유된 자원에서 동기화 대상이 하나 이상인 경우`
    - 상호 배제를 위한 동기화 기법
    - `바쁜 대기 발생`
        - Mutex Lock 과 동일
        - 해결 방법
            - 사용할 수 있는 자원이 없을 경우 대기 상태로 만듬
            (해당 프로세스의 PCB를 대기 큐에 삽입)
            - 사용할 수 있는 자원이 생겼을 경우 대기 큐의 프로세스를 준비 상태로 만듬
            (해당 프로세스의 PCB를 대기 큐에서 꺼내 준비 큐에 삽입)
    - 구현 방법
        - `열쇠 역할`
            - 전역 변수 S
        - `임계 구역 진입 여부를 알려주는 역할`
            - wait 함수
        - `임계 구역 진입 하라는 역할`
            - signal 함수
    - 종류
        - 이진 세마포
        - 카운팅 세마포

- `모니터 (Monitor)`
    - Semaphore 의 단점을 극복하기 위해 등장
        - 임계 구역에서 작업을 할 때 매번 `wait`,`signal` 함수를 호출해야 하는데 이 부분에서
        휴먼 에러가 발생할 수 있음
    - `상호 배제를 위한 동기화`
        - 객체지향 프로그래밍에서 사용되는 방법
        - 인터페이스를 위한 큐
        - 공유자원에 접근하고자 하는 프로세스를 큐에 삽입
        - 큐에 삽입된 순서대로 공유 자원 이용
        - 프로세스는 반드시 인터페이스를 통해서만 공유 자원에 접근
    - `실행 순서 제어를 위한 동기화`
        - 조건 변수를 이용하여 실행 순서 제어
        - 조건 변수와 모니터는 별개의 개념
            - 프로세스/스레드의 실행 순서를 제어하기 위해 사용하는 특별한 변수
        - 조건 변수 큐는 상호 배제를 위한 큐와는 별개의 큐

## 멀티코어 에서는 어떻게 동기화를 할 수 있을까?

- 하드웨어적으로 구현되는 것이 보편적임
- `메모리 장벽 (Memory Barriers)`
    - 메모리의 모든 변경 사항을 다른 모든 프로세스로 전파하는 명령어
- `하드웨어 명령어 (Hardware Instructions)`
    - word의 내용을 원자적으로 교환할 수 있는 특별한 하드웨어 명령어
        - 인터럽트 되지 않는 하나의 단위
    - `test_and_set()`, `compare_and_swap()` 이 대표적
- `원자적 변수 (Atomic Variables)`
    - 단일 변수에 대한 데이터 경쟁에 대해 상호 배제를 보장하는데 사용
