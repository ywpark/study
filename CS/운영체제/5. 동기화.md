# 동기화

---

## 동기화란 무엇인가

- 동기화는 여러 프로세스 또는 스레드가 공유 자원을 올바르게 사용하도록 실행 순서를 조정하는 것이다.
- 동기화의 목적은 크게 두 가지이다.
  - `실행 순서 제어`
  - `상호 배제`

## 공유 자원과 임계 구역

- `공유 자원`
  - 여러 실행 흐름이 함께 사용하는 자원
- `임계 구역`
  - 동시에 접근하면 문제가 발생할 수 있는 코드 영역
- 임계 구역 보호에 실패하면 `경쟁 조건 (Race Condition)`이 발생한다.

## 임계 구역 문제의 해결 조건

- `상호 배제`
  - 한 번에 하나의 실행 흐름만 임계 구역에 들어갈 수 있어야 한다.
- `진행`
  - 아무도 임계 구역을 사용하지 않는다면, 진입하려는 실행 흐름이 들어갈 수 있어야 한다.
- `유한 대기`
  - 특정 실행 흐름이 무한정 기다리지 않아야 한다.

## 바쁜 대기 (Busy Waiting)

- 락이 풀렸는지 계속 확인하면서 CPU를 낭비하는 방식이다.
- 구현은 단순하지만 비효율적일 수 있다.

## 뮤텍스 락 (Mutex)

- 한 번에 하나만 자원에 접근하도록 하는 락이다.
- 보통 소유 개념이 있어서 락을 획득한 실행 흐름만 해제할 수 있다.
- 상호 배제를 구현할 때 자주 사용된다.

## 세마포어 (Semaphore)

- 접근 가능한 자원의 개수를 정수 값으로 관리하는 동기화 도구이다.
- `wait()`는 자원을 요청하는 연산이고, `signal()`은 자원을 반납하는 연산이다.
- `이진 세마포어`
  - 값이 0 또는 1로 동작하는 세마포어
- `카운팅 세마포어`
  - 여러 개의 동일 자원을 관리할 수 있는 세마포어

## 뮤텍스와 세마포어의 차이

- `뮤텍스`
  - 보통 소유자가 존재한다.
  - 상호 배제 목적이 더 분명하다.
- `세마포어`
  - 소유 개념이 반드시 있지는 않다.
  - 여러 개 자원 관리와 실행 순서 제어에도 활용할 수 있다.
- 이진 세마포어와 뮤텍스는 비슷해 보이지만 의미와 사용 방식이 완전히 같지는 않다.

## 모니터 (Monitor)

- 동기화를 더 안전하게 사용하기 위해 등장한 고수준 추상화이다.
- 공유 자원에 접근하는 코드를 하나의 인터페이스 안에 넣고, 내부적으로 상호 배제를 보장한다.
- 조건 변수를 통해 실행 순서를 제어할 수 있다.

## 스핀락과 블로킹

- `스핀락`
  - 락이 풀릴 때까지 계속 확인한다.
  - 짧게 잠기는 자원에서는 유리할 수 있다.
- `블로킹 락`
  - 사용할 수 없으면 대기 상태로 전환한다.
  - CPU 낭비를 줄일 수 있다.

## 멀티코어 환경에서의 동기화

- 멀티코어에서는 여러 코어가 동시에 같은 메모리에 접근할 수 있으므로 동기화가 더 중요하다.
- 이를 위해 다음 개념이 사용된다.
  - 원자적 명령어
  - 메모리 장벽
  - 원자적 변수

## 정리

- 동기화는 공유 자원을 안전하게 다루기 위한 핵심 개념이다.
- 멀티스레드 환경에서는 동기화가 필수이며, 뮤텍스, 세마포어, 모니터의 차이를 구분해서 이해하는 것이 중요하다.
