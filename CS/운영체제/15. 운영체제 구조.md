# 운영체제 구조

---

## 기본 정보

- 운영체제는 하나의 거대한 프로그램처럼 보이지만, 내부적으로는 여러 기능이 구조적으로 나뉘어 있다.
- 운영체제를 어떤 방식으로 설계하느냐에 따라 성능, 안정성, 확장성이 달라진다.

## 운영체제 구조가 중요한 이유

- 시스템을 관리하기 쉽게 만들기 위해
- 기능 추가와 유지보수를 쉽게 하기 위해
- 안정성과 보안성을 높이기 위해
- 하드웨어와 소프트웨어 사이의 역할을 명확히 하기 위해

## 단순 구조

- 초기 운영체제에서 볼 수 있는 구조
- 기능 구분이 명확하지 않고 전체가 하나처럼 동작
- 구현은 단순하지만 유지보수가 어려움

## 모놀리식 커널 (Monolithic Kernel)

- 운영체제의 주요 기능이 모두 하나의 커널 안에서 동작하는 구조
- 예시 기능
    - 프로세스 관리
    - 메모리 관리
    - 파일 시스템
    - 장치 드라이버
- 장점
    - 성능이 좋을 수 있음
    - 커널 내부 기능 간 호출이 빠름
- 단점
    - 하나의 오류가 전체 시스템에 큰 영향을 줄 수 있음
    - 구조가 커질수록 복잡성이 증가

## 마이크로커널 (Microkernel)

- 최소한의 핵심 기능만 커널에 두고, 나머지 서비스는 사용자 공간에서 실행하는 구조
- 커널에 남는 대표 기능
    - 기본적인 스케줄링
    - 메모리 보호
    - 프로세스 간 통신 (Inter-Process Communication, IPC)
- 장점
    - 안정성과 보안성이 높음
    - 모듈 교체와 유지보수에 유리
- 단점
    - 사용자 공간과 커널 공간 사이의 통신이 많아져 성능 오버헤드가 발생할 수 있음

## 계층형 구조 (Layered Structure)

- 운영체제를 여러 계층으로 나누고, 각 계층이 바로 아래 계층의 서비스만 사용하는 구조
- 장점
    - 설계와 검증이 비교적 쉬움
    - 역할이 명확함
- 단점
    - 계층을 엄격히 나누기 어려운 경우가 있음
    - 계층을 많이 거치면 성능 손실 가능

## 모듈형 구조 (Modular Structure)

- 핵심 커널은 유지하면서 필요한 기능을 모듈 형태로 동적으로 추가/제거하는 구조
- 현대 운영체제에서 널리 사용되는 방식
- 장점
    - 유연성
    - 확장성
    - 비교적 좋은 성능

## 구조별 비교

- `모놀리식 커널`
    - 성능 중심
    - 구조 복잡도 높음
- `마이크로커널`
    - 안정성, 유지보수성 중심
    - IPC 오버헤드 존재
- `계층형 구조`
    - 역할 분리가 명확
    - 설계 제약이 있음
- `모듈형 구조`
    - 유연성과 성능의 균형

## 정리

- 운영체제 구조는 단순한 분류가 아니라, 성능과 안정성 사이의 trade-off 를 반영한 설계 선택이다.
- 구조를 이해하면 운영체제를 단순한 기능 집합이 아니라 설계된 시스템으로 볼 수 있다.
