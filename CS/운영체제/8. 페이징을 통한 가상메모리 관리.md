# 페이징을 통한 가상메모리 관리

---

## 가상 메모리란 무엇인가

- 가상 메모리는 프로그램이 실제 물리 메모리보다 더 큰 메모리를 사용하는 것처럼 보이게 해주는 기술이다.
- 운영체제는 일부만 메모리에 올리고 나머지는 디스크에 두면서, 필요한 순간에만 적재한다.

## 페이징의 기본 개념

- 가상 주소 공간을 일정한 크기의 `페이지`로 나눈다.
- 물리 메모리는 같은 크기의 `프레임`으로 나눈다.
- 운영체제는 각 페이지를 빈 프레임에 배치하여 메모리를 관리한다.

## 페이지 테이블

- 페이지 번호와 프레임 번호의 대응 관계를 저장하는 자료구조이다.
- 프로세스마다 별도의 페이지 테이블을 가진다.
- 덕분에 프로세스 입장에서는 메모리가 연속된 것처럼 보인다.

## 논리 주소와 물리 주소 변환

- CPU가 생성하는 주소는 `논리 주소`이다.
- 논리 주소는 보통 다음 두 부분으로 구성된다.
  - `페이지 번호`
  - `오프셋`
- 운영체제는 페이지 번호를 통해 페이지 테이블을 조회하여 프레임 번호를 찾고, 여기에 오프셋을 더해 실제 물리 주소를 계산한다.

## 페이지 테이블 기준 레지스터와 TLB

- `페이지 테이블 기준 레지스터 (Page Table Base Register, PTBR)`
  - 현재 실행 중인 프로세스의 페이지 테이블 시작 주소를 가리키는 레지스터
- 주소 변환을 위해 매번 페이지 테이블에 접근하면 메모리 접근이 늘어난다.
- 이를 줄이기 위해 `변환 색인 버퍼 (Translation Lookaside Buffer, TLB)`를 사용한다.
- `변환 색인 버퍼 (Translation Lookaside Buffer, TLB)`
  - 최근 사용한 페이지 테이블 엔트리를 캐시하는 고속 메모리
- `TLB hit`
  - TLB에서 바로 주소 변환 정보를 찾은 경우
- `TLB miss`
  - TLB에 정보가 없어 페이지 테이블까지 가야 하는 경우

## 페이지 테이블 엔트리 (Page Table Entry, PTE)

- 페이지 테이블의 각 항목이다.
- 일반적으로 다음 정보가 들어간다.
  - 유효 비트
  - 보호 비트
  - 참조 비트
  - 수정 비트
  - 프레임 번호

## 페이지 폴트 처리 과정

1. CPU가 특정 페이지에 접근한다.
2. 해당 페이지가 메모리에 없으면 페이지 폴트가 발생한다.
3. 운영체제가 페이지 폴트 처리 루틴을 수행한다.
4. 필요한 페이지를 보조기억장치에서 메모리로 적재한다.
5. 페이지 테이블을 갱신한 뒤 중단된 명령을 다시 실행한다.

## 내부 단편화

- 페이징은 외부 단편화를 줄일 수 있지만, 페이지 크기에 딱 맞지 않는 마지막 부분에서 내부 단편화가 발생할 수 있다.
- 다만 일반적으로 그 크기는 비교적 작다.

## 세그멘테이션과 비교

- `페이징`
  - 고정 크기 단위
  - 외부 단편화 완화
  - 논리적 의미보다는 관리 효율 중심
- `세그멘테이션`
  - 가변 크기 단위
  - 코드, 데이터, 스택처럼 논리적 단위로 나눔
  - 외부 단편화 가능성 존재

## 계층적 페이지 테이블

- 페이지 테이블이 너무 커질 수 있기 때문에, 페이지 테이블 자체를 여러 단계로 나누는 방법이다.
- 메모리 사용량을 줄이는 데 도움이 되지만 주소 변환 과정은 더 복잡해질 수 있다.

## Copy On Write

- `fork()` 직후 부모와 자식이 같은 물리 프레임을 공유하게 하고, 실제 쓰기 시점에만 복사하는 방식이다.
- 불필요한 메모리 복사를 줄여 성능과 메모리 효율을 높인다.

## 정리

- 페이징은 현대 운영체제 메모리 관리의 핵심이며, 큰 프로그램 실행과 효율적인 메모리 사용을 가능하게 한다.
- 페이지 테이블, TLB, 페이지 폴트, 계층적 페이지 테이블은 함께 이해해야 하는 연결된 개념이다.
