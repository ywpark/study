# 메모리 관리 심화

---

## 기본 정보

- 운영체제의 메모리 관리는 단순히 공간을 나누는 것을 넘어, 프로그램이 메모리를 안전하고 효율적으로 사용할 수 있도록 하는 과정이다.
- 이 챕터는 연속 메모리 할당, 페이징, 세그멘테이션을 잇는 상위 개념을 정리한다.

## 주소 바인딩 (Address Binding)

- 프로그램의 주소를 실제 메모리 주소와 연결하는 과정
- 시점에 따라 나뉨
    - `컴파일 시간 바인딩`
        - 물리 주소가 미리 결정되는 경우
    - `적재 시간 바인딩`
        - 메모리에 올라갈 때 주소 결정
    - `실행 시간 바인딩`
        - 실행 중 동적으로 주소 변환
- 현대 운영체제는 주로 실행 시간 바인딩을 사용

## 논리 주소와 물리 주소

- `논리 주소 (Logical Address)`
    - CPU가 생성하는 주소
    - 프로세스 입장에서 보이는 주소
- `물리 주소 (Physical Address)`
    - 실제 메모리 상의 주소
- 운영체제와 하드웨어는 논리 주소를 물리 주소로 변환해 사용

## 메모리 관리 장치 (Memory Management Unit, MMU)

- 논리 주소를 물리 주소로 변환하는 하드웨어 장치
- CPU와 메모리 사이에서 주소 변환 담당
- 가상 메모리 구현의 핵심 구성 요소

## 동적 적재 (Dynamic Loading)

- 프로그램 전체를 한 번에 메모리에 올리지 않고, 필요한 부분만 적재하는 방식
- 메모리 절약 가능
- 사용하지 않는 루틴은 끝까지 적재되지 않을 수 있음

## 동적 연결 (Dynamic Linking)

- 라이브러리를 프로그램 실행 시점에 연결하는 방식
- 실행 파일 크기를 줄일 수 있음
- 여러 프로세스가 같은 라이브러리를 공유하기 쉬움

## 세그멘테이션 심화

- 프로그램을 논리적인 단위로 나누어 관리하는 방식
- 세그먼트 예시
    - 코드
    - 데이터
    - 스택
    - 힙
- 장점
    - 논리 구조를 반영하기 쉬움
    - 보호와 공유에 유리
- 단점
    - 가변 크기이므로 외부 단편화 발생 가능

## 페이징과 세그멘테이션 비교

- `페이징`
    - 고정 크기
    - 외부 단편화 해결에 유리
    - 관리 효율 중심
- `세그멘테이션`
    - 가변 크기
    - 프로그램의 논리 구조 반영
    - 외부 단편화 발생 가능

## 메모리 보호

- 한 프로세스가 다른 프로세스의 메모리에 함부로 접근하면 안됨
- 이를 위해 운영체제는 다음과 같은 보호 장치를 사용
    - 사용자 모드 / 커널 모드
    - base / limit register
    - 페이지 보호 비트

## 정리

- 메모리 관리 심화 개념은 개별 기법을 따로 외우는 것이 아니라, 주소 변환과 보호라는 큰 흐름 속에서 이해하는 것이 중요하다.
- 이 관점을 가지면 페이징, 세그멘테이션, 가상 메모리를 더 자연스럽게 연결할 수 있다.
