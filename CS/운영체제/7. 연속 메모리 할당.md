# 연속 메모리 할당

---

## 기본 정보

- 프로세스에 물리적 메모리의 한 곳에 연속적으로 적재하는 방식

## 스와핑 (Swapping)

- 주기억장치(메모리)에 적재한 하나의 프로세스를 보조기억장치에 잠시 적재했다가
필요할 때 다시 꺼내서 사용하는 메모리를 교체하는 기법
- `스왑 인 (Swap in)`
    - 보조기억장치 → 메모리로 올려서 실행 시키는 작업
- `스왑 아웃 (Swap out)`
    - 메모리 → 보조기억장치로 옮기는 작업
- 스와핑을 왜 하는 걸까?
    - 한정된 물리적 메모를 효율적으로 사용하고, 다중 프로그래밍을 가능하게 하기 위해서

## 프로세스의 메모리 배치 기법

- 프로세스는 메모리의 빈 공간에 할당되는데 빈 공간이 여러 개 있다면 어떻게 OS는
할당할까?
- `최초 적합 (First Fit)`
    - OS 가 메모리 내의 빈 공간을 순서대로 검색하다 적재할 수 있는 공간을 발견하면
    그 공간에 배치
    - 검색 최소화
    - 빠른 할당
- `최적 적합 (Best Fit)`
    - OS 가 빈 공간을 모두 검색해본 뒤 적재 가능한 가장 작은 공간에 배치
- `최악 적합 (Worst Fit)`
    - OS 가 빈 공간을 모두 검색해본 뒤 적재 가능한 가장 큰 공간에 배치

## 외부 단편화 (External Fragmentation)

- 프로세스를 연속적으로 메모리에 할당하는 방식은 메모리를 효율적으로 사용하는 방법이 아님
- `외부 단편화 라는 문제가 발생`
- 프로세스들이 실행 → 종료가 반복되면서 메모리 사이 사이에 빈 공간이 발생
- 프로세스를 할당하기 어려울 만큼 작은 메모리 공간들로 인해 메모리가 낭비되는 현상
- 해결 방법
    - `압축 (Compaction)`
        - 메모리 내에 흩어져 있는 빈 공간들을 하나로 모으는 방식
        - 프로세스를 재배치하여 흩어져 있는 작은 공간들을 한쪽으로 모아서 하나의 큰 연속적인
        공간을 만듬
    - `페이징 (Paging)`
        - 가상메모리를 활용
