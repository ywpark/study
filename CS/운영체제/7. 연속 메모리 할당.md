# 연속 메모리 할당

---

## 메모리 관리가 필요한 이유

- 여러 프로세스가 동시에 실행되면 메모리에 각 프로세스를 적절히 배치해야 한다.
- 운영체제는 한정된 물리 메모리를 효율적으로 사용하면서도 각 프로세스를 서로 보호해야 한다.

## 연속 메모리 할당 방식

- 하나의 프로세스를 물리 메모리의 연속된 공간에 배치하는 방식이다.
- 구현은 직관적이지만, 시간이 지날수록 메모리 공간이 조각나기 쉽다.

## 고정 분할 방식과 가변 분할 방식

- `고정 분할`
  - 메모리를 미리 고정된 크기의 여러 영역으로 나눈다.
  - 관리가 단순하지만 `내부 단편화`가 발생하기 쉽다.
- `가변 분할`
  - 프로세스 크기에 맞게 메모리를 나누어 할당한다.
  - 내부 단편화는 줄일 수 있지만 `외부 단편화`가 발생하기 쉽다.

## 스와핑 (Swapping)

- 메모리에 있는 프로세스를 보조기억장치로 내보내고, 다른 프로세스를 메모리에 적재하는 기법이다.
- `스왑 아웃`
  - 메모리에서 디스크로 내보내는 작업
- `스왑 인`
  - 디스크에서 메모리로 다시 가져오는 작업
- 메모리가 부족할 때 더 많은 프로세스를 동시에 다루기 위해 사용된다.

## 메모리 배치 전략

- 빈 공간이 여러 개일 때 어떤 위치에 프로세스를 넣을지 결정하는 방법이다.
- `First Fit`
  - 처음 발견한 적절한 공간에 할당
- `Best Fit`
  - 들어갈 수 있는 공간 중 가장 작은 곳에 할당
- `Worst Fit`
  - 들어갈 수 있는 공간 중 가장 큰 곳에 할당

## 내부 단편화와 외부 단편화

- `내부 단편화`
  - 할당된 공간 내부에 사용되지 않는 공간이 남는 현상
- `외부 단편화`
  - 빈 공간이 여러 군데 흩어져 있어 전체 여유 공간은 충분하지만 큰 프로세스를 넣지 못하는 현상
- 연속 메모리 할당에서는 특히 외부 단편화가 큰 문제가 된다.

## 압축 (Compaction)

- 흩어진 빈 공간을 한쪽으로 모아 큰 연속 공간으로 만드는 방법이다.
- 효과는 있지만, 프로세스를 옮겨야 하므로 비용이 크다.

## 연속 할당 방식의 한계

- 외부 단편화 문제
- 큰 프로세스를 적재하기 어려움
- 메모리 재배치 비용 발생
- 효율적인 다중 프로그래밍에 한계가 있음

## 페이징이 등장한 이유

- 연속 메모리 할당의 한계를 해결하기 위해 프로세스를 꼭 연속된 공간에 넣지 않아도 되는 방식이 필요했다.
- 그 결과 등장한 대표적 기법이 `페이징`이다.

## 정리

- 연속 메모리 할당은 이해하기 쉬운 방식이지만, 단편화 문제 때문에 현대 운영체제에서는 주로 가상 메모리와 페이징 기법이 사용된다.
- 따라서 이 챕터는 페이징이 왜 필요한지 설명하는 출발점으로 이해하면 좋다.
