# 프로세스

---

## 기본 정보

- `프로세스 (Process)`
    - 컴퓨터에서 실행 중인 프로그램
- 종류
    - `포그라운드 프로세스 (Foreground Process)`
        - 사용자와 상호 작용하는 프로세스
    - `백그라운드 프로세스 (Background Process)`
        - 사용자와의 직접적인 상호 작용 없이 실행되는 프로세스
        - `데몬(Daemon)`, `서비스(Service)` 라고도 불림

## 프로세스 제어 블록 (Process Control Block)

- 배경
    - 모든 프로세스는 실행을 위해서 CPU 가 필요
        - 프로세스들은 돌아가며 한정된 시간 만큼만 CPU를 사용
- 정의
    - 프로세들을 관리하기 위해 사용하는 자료구조
- 프로세스 관련 정보를 저장하는 태그와 같은 정보
- `프로세스 생성 시 커널 영역에 생성, 종료 시 폐기`
- 운영체제마다 PCB 에 저장되는 정보가 다름
- 기본적인 PCB 정보
    - 프로세스 ID
        - 특정 프로세스를 식별하기 위해 부여하는 고유한 번호
    - 레지스터 값
        - 프로세스는 실행 차례가 오면 이전까지 사용한 레지스터 값을 모두 복원 후
        실행 재개
        - `프로그램 카운터(PC)` , `스택포인터`
    - 프로세스 상태
        - 입출력 장치를 사용하기 위해 기다리는 상태
        - CPU를 사용하기 위해 기다리는 상태
        - CPU를 이용 중인 상태
    - CPU 스케줄링 정보
        - 프로세스가 언제, 어떤 순서로 CPU를 할당 받을지에 대한 정보
    - 메모리 정보
        - 프로세스가 어느 주소에 저장되어 있는지에 대한 정보
        - 페이지 테이블 정보
    - 사용한 파일과 입출력장치 정보
        - 할당된 입출력 장치
        - 사용 중인 파일 정보
- `문맥 교환 (Context Switch)`
    - 실행 중인 프로세스 Context 를 백업하고 새로운 프로세스 실행을 위해 Context를 
    복구하는 과정
    - `다른 프로세스로 실행 순서가 변경된다면?`
        - 기존 프로세스의 Context 를 백업
        - 실행할 프로세스의 Context 를 복구

## 프로세스 메모리 영역

- PCB는 커널영역에 저장되고 추가로 프로세스를 메모리의 사용자영역에 저장
- 종류
    - `코드 (Code) 영역`
        - 기계어로 이루어진 명령어 저장
        - CPU가 실행할 명령어가 저장되기 때문에 `Read-Only`
        - 정적 할당 영역
    - `데이터 (Data) 영역`
        - 프로그램이 사용하는 데이터를 저장
        - `전역 변수`, `static 변수`
        - 정적 할당 영역
    - `힙 (Heap) 영역`
        - 프로그래머가 직접 할당할 수 있는 영역
        - 동적 할당 영역
    - `스택 (Stack) 영역`
        - 데이터가 임시로 저장되는 영역
        - `복귀할 주소`, `데이터 (지역변수, 매개변수, 리턴값 등)`
        - 동적 할당 영역

- 정의
    - `생성 상태`
        - 메모리에 적재되어 PCB를 할당 받은 상태
        - 준비가 완료되면 준비 상태로 전환
    - `준비 상태`
        - CPU를 할당 받아 실행할 수 있지만 자신의 차례를 기다리는 상태
        - 자신의 차례가 되면 실행 상태로 전환 ( Dispatch )
    - `실행 상태`
        - CPU를 할당 받아 실행 중인 상태
        - 할당된 시간이 지나면 준비 상태로 전환 ( Pause )
        - 실행 도중 입출력장치를 사용하면 입출력 작업이 끝날 때까지 대기 상태로 전환
    - `대기 상태`
        - 프로세스가 실행 도중 입출력 장치를 사용하는 경우
        - 입출력 작업은 CPU에 비해 느리기 때문에 대기 상태로 전환됨
        - 입출력 작업이 끝나면 준비 상태로 전환
            - 입출력 완료 인터럽트 신호를 받으면 끝남
    - `종료 상태`
        - 프로세스가 종료된 상태
        - PCB, 프로세스 메모리 영역 정리

## 프로세스 계층 구조

- 프로세스 실행 도중에 시스템 콜을 통해 다른 프로세스 생성 가능
- `부모 프로세스`
    - 프로세스를 생성한 주체
- `자식 프로세스`
    - 부모 프로세스에 의해 생성된 프로세스
- `fork 시스템 콜`
    - 부모 프로세스가 자신의 복사본을 자식 프로세스 생성
    - 프로세스 ID 가 별도로 생성
    - 부모 프로세스의 자원 상속
- `exec 시스템 콜`
    - 자식 프로세스가 자신의 메모리 공간을 다른 프로그램으로 교체
    - 메모리 공간을 새로운 프로그램으로 덮어쓰기
    - 코드/데이터 영역은 실행할 프로그램 내용으로 바뀌고 나머지 영역은 초기화
