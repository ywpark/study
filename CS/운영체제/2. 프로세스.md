# 프로세스

---

## 프로그램과 프로세스의 차이

- `프로그램 (Program)`
  - 디스크에 저장된 실행 파일 그 자체이다.
- `프로세스 (Process)`
  - 메모리에 적재되어 CPU의 실행 대상이 된 프로그램이다.
- 즉, 프로그램은 정적인 개념이고 프로세스는 실행 중인 동적인 개념이다.

## 프로세스의 구성 요소

- 프로세스는 단순한 코드 덩어리가 아니라 다음 요소를 함께 가진다.
  - 코드 영역
  - 데이터 영역
  - 힙 영역
  - 스택 영역
  - `프로세스 제어 블록 (Process Control Block, PCB)`

## 프로세스 제어 블록 (PCB)

- 운영체제가 각 프로세스를 관리하기 위해 유지하는 자료구조이다.
- PCB에는 다음과 같은 정보가 저장된다.
  - 프로세스 ID
  - 프로세스 상태
  - 프로그램 카운터
  - 레지스터 값
  - 메모리 관련 정보
  - 중앙처리장치 스케줄링 정보
  - 열린 파일과 입출력 장치 정보
- PCB는 커널 영역에 존재하며, 운영체제가 프로세스를 식별하고 다시 실행하기 위한 핵심 정보가 된다.

## 문맥 교환 (Context Switch)

- CPU가 한 프로세스에서 다른 프로세스로 실행 대상을 바꾸는 과정이다.
- 이때 현재 프로세스의 상태를 PCB에 저장하고, 다음 프로세스의 상태를 PCB에서 복원한다.
- 문맥 교환은 CPU가 실제 작업을 하는 시간이 아니라 준비 작업에 가까우므로 오버헤드가 발생한다.
- 문맥 교환 비용이 큰 이유
  - 레지스터 저장 및 복원 필요
  - 캐시 활용성이 떨어질 수 있음
  - 커널 모드 전환이 수반될 수 있음

## 프로세스 메모리 구조

- `코드 영역`
  - 실행할 명령어가 저장되는 영역
  - 일반적으로 읽기 전용이다.
- `데이터 영역`
  - 전역 변수와 static 변수가 저장된다.
- `힙 영역`
  - 동적 메모리 할당에 사용된다.
- `스택 영역`
  - 함수 호출 정보, 지역 변수, 매개변수 등이 저장된다.

## 프로세스 상태

- `생성`
  - 프로세스가 만들어지는 상태
- `준비`
  - CPU를 받기만 하면 실행 가능한 상태
- `실행`
  - CPU를 할당받아 실제로 실행 중인 상태
- `대기`
  - 입출력 완료 같은 특정 사건을 기다리는 상태
- `종료`
  - 실행을 마친 상태

## 부모 프로세스와 자식 프로세스

- 운영체제는 실행 중인 프로세스가 또 다른 프로세스를 생성할 수 있게 한다.
- 프로세스를 생성한 쪽을 `부모 프로세스`, 생성된 쪽을 `자식 프로세스`라고 한다.

## `fork()`와 `exec()`

- `fork()`
  - 현재 프로세스를 복제하여 새로운 자식 프로세스를 만든다.
  - 부모와 자식은 거의 같은 상태로 시작하지만 PID는 다르다.
- `exec()`
  - 현재 프로세스의 메모리 공간을 새로운 프로그램으로 덮어쓴다.
  - 보통 자식 프로세스가 `fork()` 이후 `exec()`를 호출하여 다른 프로그램을 실행한다.
- 두 시스템 콜이 분리되어 있기 때문에 프로세스 생성과 새 프로그램 실행을 유연하게 조합할 수 있다.

## 좀비 프로세스와 고아 프로세스

- `좀비 프로세스 (Zombie Process)`
  - 실행은 끝났지만 부모 프로세스가 종료 상태를 회수하지 않아 PCB 정보가 남아 있는 프로세스
- `고아 프로세스 (Orphan Process)`
  - 부모 프로세스가 먼저 종료되어 보호자가 없어진 자식 프로세스
- 유닉스 계열 시스템에서는 보통 init 또는 유사한 프로세스가 고아 프로세스를 수거한다.

## 정리

- 프로세스는 단순한 실행 파일이 아니라, 메모리 구조와 PCB를 가진 운영체제의 관리 단위이다.
- 프로세스 관리를 이해하면 CPU 스케줄링, 문맥 교환, 메모리 관리, 스레드 개념을 더 쉽게 연결할 수 있다.
